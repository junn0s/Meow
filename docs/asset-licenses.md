# 에셋 및 라이선스

## 1. 요약

이 프로젝트는 외부 이미지, 스프라이트 시트, 타일맵, 아이콘, 폰트 파일, 배경음악, 효과음 파일을 사용하지 않습니다. 게임에 보이는 픽셀아트와 들리는 효과음은 프로젝트 소스 코드가 브라우저에서 직접 생성합니다.

따라서 제출물에 별도로 첨부하거나 저작자를 표시해야 하는 제3자 미디어 에셋은 없습니다.

| 구분 | 외부 파일 사용 | 생성 방식 | 별도 에셋 라이선스 |
| --- | --- | --- | --- |
| 캐릭터 | 없음 | Phaser Graphics/Canvas의 사각 픽셀 조합 | 해당 없음 |
| 음식·가구·간판 | 없음 | 코드로 색상과 도형을 그려 텍스처 생성 | 해당 없음 |
| 배경·비·김·하트·동전 효과 | 없음 | 도형, 반복 패턴, 파티클을 런타임 생성 | 해당 없음 |
| UI 아이콘 | 없음 | 텍스트와 코드 생성 도형 | 해당 없음 |
| 파비콘 | 외부 에셋 없음 | 프로젝트에서 직접 작성한 `public/favicon.svg` 도형 | 제3자 라이선스 해당 없음 |
| 효과음 | 없음 | Web Audio API oscillator/gain/envelope 합성 | 해당 없음 |
| 배경음악 | 사용하지 않음 | 해당 없음 | 해당 없음 |
| 웹폰트 | 사용하지 않음 | 브라우저·운영체제 기본 폰트 사용 | 파일 배포 없음 |

## 2. 그래픽 생성 방식

캐릭터와 오브젝트는 작은 캔버스 또는 Phaser Graphics에 정수 좌표와 제한된 색상 팔레트로 도형을 그린 뒤 게임 텍스처로 등록합니다. 생성되는 대상은 다음과 같습니다.

- 치즈 고양이 사장, 4색 모자띠 턱시도 고양이 셰프, 4색 배지 삼색 고양이 서버
- 토끼, 강아지, 햄스터, 너구리 손님
- 어묵, 떡볶이, 붕어빵, 야식 라면, 달빛 꼬치, 달빛 정식
- 포장마차, 조리대, 냄비, 테이블, 좌석, 네온·달빛 간판
- 주문 말풍선, 인내심 게이지, 별점, 하트, 동전
- 낮·노을·밤·새벽 거리 배경, 빗방울, 안개, 젖은 바닥 반사, 음식의 김과 24개 조명 효과

이 방식은 외부 PNG, GIF, SVG 또는 스프라이트 시트를 복사하거나 변환하지 않습니다. 그래픽은 프로젝트 전용 코드와 기획에 따라 독자적으로 구성합니다. 브라우저 탭의 `public/favicon.svg` 역시 외부 파일이 아니라 같은 팔레트와 단순 도형으로 프로젝트에서 직접 작성한 벡터입니다. 파일의 W3C SVG namespace 문자열은 형식 식별자이며 외부 네트워크 요청이 아닙니다.

픽셀 표현은 `480 × 270` 논리 좌표계를 `960 × 540` 실제 버퍼에 렌더링하고, Phaser의 `pixelArt: true`, `roundPixels: true`, nearest-neighbor 필터를 사용합니다. 글자는 별도의 2배 내부 해상도로 그리며, 이 과정에서 새로운 외부 에셋이 추가되지 않습니다.

## 3. 사운드 생성 방식

효과음은 Web Audio API로 실행 시 합성합니다. oscillator의 파형·주파수 변화, gain envelope, 짧은 노이즈를 조합해 다음 신호를 만듭니다.

- 손님 입장
- 음식 완성
- 동전 획득
- 구매 성공
- 업그레이드 해금
- 게임 클리어
- 낮·노을·밤·새벽에 따라 주파수가 부드럽게 바뀌는 저음 앰비언스

MP3, WAV, OGG 등 녹음·샘플 파일을 포함하지 않으며 외부 음원에서 일부를 추출하지 않습니다. 브라우저 자동 재생 정책을 지키기 위해 첫 키 입력 또는 클릭 후 오디오 컨텍스트를 활성화합니다.

## 4. 글꼴과 문자

프로젝트는 웹폰트 파일을 내려받거나 저장소에 포함하지 않습니다. CSS의 시스템 폰트 스택을 사용하므로 실제 표시 글꼴은 플레이어의 운영체제와 브라우저가 제공합니다. 게임 그래픽을 대체하기 위한 외부 이모지 이미지도 배포하지 않습니다.

## 5. 소프트웨어 의존성과의 구분

Phaser, TypeScript, Vite와 개발 도구는 게임 제작·빌드를 위한 오픈 소스 소프트웨어 의존성입니다. 이들은 본 문서의 미디어 에셋이 아니며, 설치 시 각 패키지의 라이선스가 적용됩니다. 정확한 버전은 `package.json`과 잠금 파일을 기준으로 확인합니다.

대표 라이선스는 다음과 같으며, 제출 전 잠금 파일에 설치된 모든 직접·간접 의존성의 라이선스를 다시 확인해야 합니다.

| 소프트웨어 | 용도 | 대표 라이선스 |
| --- | --- | --- |
| Phaser | 게임 런타임 | MIT |
| TypeScript | 정적 타입 검사·컴파일 | Apache-2.0 |
| Vite | 개발 서버·프로덕션 빌드 | MIT |

라이선스의 최종 기준은 설치된 패키지에 포함된 원문과 공식 배포 정보입니다.

## 6. AI 사용과 권리 검토

Codex는 게임 코드, 문서, 코드 기반 도형 구성과 합성음 파라미터 작성에 사용됩니다. 특정 상용 게임, 캐릭터, 로고, 작가의 그림 또는 음원을 복제하도록 요청하지 않았습니다. 결과물은 사람이 기획과 코드를 검토하고 프로젝트 전용 표현으로 조정합니다.

AI를 사용했다는 사실은 [AI 활용 보고서](ai-usage-report.md)에 별도로 공개합니다. AI 사용 자체를 제3자 에셋의 출처 표기를 대신하는 수단으로 사용하지 않습니다.

## 7. 외부 에셋 추가 시 절차

현재 제출본에는 외부 에셋이 없지만, 이후 추가할 경우 다음 절차를 따릅니다.

1. 원본 다운로드 URL, 제작자, 에셋명, 버전과 다운로드 날짜를 기록합니다.
2. 상업적 이용, 수정, 재배포, 저작자 표시 조건을 확인합니다.
3. 라이선스 원문 또는 링크를 저장하고 이 문서의 표에 추가합니다.
4. 금지된 용도나 출처가 불명확한 에셋은 저장소와 빌드에서 제거합니다.
5. 수정한 경우 원본과 수정 범위를 함께 기록합니다.
6. 효과음·음악은 샘플 일부만 사용하더라도 동일하게 기록합니다.

## 8. 제출 전 확인

- [ ] `public/`과 빌드 산출물에 출처 불명의 이미지·음원·폰트 파일이 없다.
- [ ] 코드 생성 텍스처가 외부 이미지를 런타임에 요청하지 않는다.
- [ ] Network 패널에 제3자 에셋 요청이 없다.
- [ ] Web Audio 효과음이 샘플 파일 없이 합성된다.
- [ ] 설치된 소프트웨어 의존성 버전과 라이선스를 잠금 파일 기준으로 확인했다.
- [ ] 새로 추가된 모든 외부 에셋이 있다면 본 문서에 출처와 라이선스를 기록했다.
