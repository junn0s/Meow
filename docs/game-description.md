# 냥포차: 달빛 아래 야식당 — 게임 설명서

## 1. 개요

| 항목 | 내용 |
| --- | --- |
| 영문명 | Meow Night Diner |
| 장르 | 경영 시뮬레이션·방치형 타이쿤·2D 픽셀아트 |
| 플랫폼 | 데스크톱·모바일 웹 브라우저 |
| 성장 길이 | 30단계, 목표 활성 플레이 381분 |
| 기술 | Phaser 3, TypeScript, Vite |
| 해상도 | 논리 480×270 / 렌더 960×540 |

비 오는 거리의 작은 포장마차를 운영해 6가지 야식을 팔고, 8명의 고양이 직원을 고용하며, 낮의 포근한 가게를 밤의 푸른 네온 명소로 성장시키는 경영 게임입니다.

## 2. 핵심 플레이 루프

```text
손님 입장 → 좌석 배정 → 주문 → 조리 → 서빙 → 식사 → 결제
        ↘ 인내심·이탈·평점 ↗
결제 수익 → 세부 구매 → 메뉴/좌석/직원/외형 성장 → 더 큰 수요
```

초반에는 사장 고양이가 직접 주문과 서빙을 담당합니다. 자동화 이후에는 어떤 메뉴, 좌석, 조리 슬롯과 서버 동선이 병목인지 판단하고 피버·홍보를 러시에 맞춰 운용합니다.

## 3. 조작과 접근성

| 입력 | 동작 |
| --- | --- |
| WASD·방향키 | 이동 |
| Space | 가까운 손님 주문, 완성 음식 들기, 일치 주문 서빙 |
| 마우스·터치 | 구매·버튼 |
| Esc | 일시정지·설정 |
| M | 음소거 |
| 모바일 패드·ACTION | 이동·상호작용 |

모바일 조작 버튼은 최소 44×44px입니다. 상태는 색에만 의존하지 않고 `VIP`, `FEVER`, `RUSH`, `!`, `!!`, 주문 수량을 글자와 형태로 표시합니다. 운영체제 `prefers-reduced-motion` 또는 일시정지 메뉴의 움직임 줄이기를 사용하면 비·안개·증기·네온 전환과 결과 화면 반복 효과가 감소합니다.

## 4. 손님과 서비스

손님 상태는 다음 순서를 따릅니다.

```text
ENTERING → WAITING_FOR_SEAT → MOVING_TO_SEAT → ORDERING
→ WAITING_FOR_FOOD → EATING → PAYING → LEAVING
```

- 토끼: 기본 인내심 `×0.8`, 빠른 대응 필요
- 햄스터: 인내심 `×0.95`
- 강아지: 인내심 `×1.1`
- 너구리: 인내심 `×1.2`, 다량 주문 확률 증가
- VIP: 23단계 이후 등장, 인내심 `×1.25`, 기본 결제 `×2`, 별도 팁 없음

인내심이 0이 되면 결제하지 않고 떠나고 콤보가 끊깁니다. 세 명이 연속 이탈하면 30초 동안 다음 손님 간격을 15% 늦추는 안전장치가 작동합니다. 평점은 최근 서비스 점수를 지수 가중 평균으로 반영하므로 예전 성공만으로 높은 평점을 영구 유지할 수 없습니다.

## 5. 메뉴와 주문

| 메뉴 | 해금 | 기본 가격 | 기본 조리시간 |
| --- | ---: | ---: | ---: |
| 어묵 | 1단계 | 18냥 | 4초 |
| 떡볶이 | 6단계 | 630냥 | 7초 |
| 붕어빵 | 11단계 | 22K냥 | 10초 |
| 야식 라면 | 16단계 | 790K냥 | 13초 |
| 달빛 꼬치 | 21단계 | 28M냥 | 16초 |
| 달빛 정식 | 26단계 | 1B냥 | 20초 |

가격 레벨과 조리 속도 레벨은 분리됩니다. 가격은 지수 성장과 10/25/50/75/100레벨 이정표 배수를 사용하고, 조리시간은 최저 35% 바닥을 둡니다. 2·3개 주문은 추가 한 개당 기본 조리시간의 65%를 더합니다.

주문 수량 분포는 5단계 구간마다 강해져 26~30단계에는 1개 55%, 2개 35%, 3개 10%가 됩니다. 새 메뉴는 해금 직후 18% 가중치로 시작해 네 단계에 걸쳐 성숙 가중치로 올라갑니다.

## 6. 직원

| 역할 | 고용 단계 | 최대 인원 | 동작 |
| --- | --- | ---: | --- |
| 셰프 | 4/12/19/27 | 4명 | 주문자 한 명을 점유하고 해당 조리 티켓 완료까지 담당 |
| 서버 | 7/14/22/28 | 4명 | 완성 티켓 하나를 꺼내 정확한 주문자에게 왕복 배달 |

조리 동시 슬롯은 `max(1, 셰프 수)`입니다. 같은 티켓은 한 번만 완성 대기열에서 제거되고, 서버가 이동 중인 손님은 플레이어 수동 서빙 대상에서 제외됩니다. 손님이 떠나면 대기·조리·완성 티켓과 직원 점유를 함께 취소합니다.

직원 1~4번은 각각 빨강·파랑·초록·금색 모자띠/앞치마 또는 배지/트레이로 구분됩니다. 고용 인원만큼 포차 작업등도 늘어납니다.

## 7. 결제·피버·홍보

```text
결제 = 메뉴 가격 × 수량 × VIP × 콤보 × FEVER/홍보
팁   = 위 기본 결제 × 팁률
```

- 콤보: 5/10/20회 연속 성공 시 `×1.05/×1.10/×1.15`
- 팁: 빠른 서비스 20%, 보통 서비스 10%, 늦은 서비스 0%
- FEVER Lv.1/2/3: 9/18/29단계 해금
- 피버 지속: 15/18/20초, 결제 `×1.5/×1.65/×1.8`
- 메뉴 홍보: 30초 동안 대상 메뉴 주문 가중치 `×2`, 결제 `×1.25`

피버는 확정 게이지 방식이며 확률 상자를 사용하지 않습니다. 레벨별 시뮬레이션 매출 기여 목표는 10~17%입니다.

## 8. 난이도와 30단계

| 구간 | 단계당 목표 | 운영 변화 |
| --- | ---: | --- |
| 1~5 | 30초~2분 | 규칙 학습, 첫 셰프 |
| 6~10 | 4분 | 떡볶이, 첫 서버, FEVER Lv.1 |
| 11~15 | 8분 | 붕어빵, 직원 2명, 다량 주문 |
| 16~20 | 12분 | 라면, FEVER Lv.2, 높은 주방 압력 |
| 21~25 | 17분 | 꼬치, VIP, 러시 시작 |
| 26~30 | 20/25/30/35/60분 | 정식, 직원 4명, 최종 복합 병목 |

25~30단계는 10초 예고 뒤 25~30초 러시가 발생합니다. 러시는 자홍색 경고와 `RUSH`, 피버는 청록색 점등과 `FEVER`로 표시됩니다. 마지막 30단계는 12분 간격의 달빛 간판 5부품으로 나뉘며 각 부품은 수익 `×1.12`와 점등 연출을 제공합니다.

## 9. 시간대와 디자인

활성 플레이 10분이 낮→노을→밤→새벽 한 주기입니다. 일시정지·튜토리얼·메뉴·오프라인 시간은 시계를 진행시키지 않습니다.

- 낮: 높은 가독성의 크림·주황 포차
- 노을: 따뜻한 하늘과 약한 반사
- 밤: 군청 도시, 청록 네온, 자홍 포인트, 젖은 바닥 반사
- 새벽: 보랏빛 하늘과 낮은 안개

밤 진입 시 최대 24개 네온이 약 1.2초 동안 순서대로 켜집니다. 분위기 파티클은 비 48개와 안개 12개로 구성한 60개 고정 풀이며 매 프레임 새 객체를 만들지 않습니다.

## 10. 오프라인과 저장

셰프와 서버가 모두 있을 때만 오프라인 수익이 발생합니다. 최대 4시간, 예상 자동 수익의 25%, 현재 다음 구매비의 45%를 상한으로 사용합니다. 돈만 지급하며 구매와 단계 진행은 자동 실행하지 않습니다.

`SaveData v3`는 돈, 평점, 누적 손님, 진행, 메뉴, 직원, 피버, 시간대, 활성 시간, 소리와 움직임 설정, 클리어 상태를 저장합니다. v1/v2 저장은 6메뉴·30단계 구조로 안전하게 보정됩니다.

## 11. 클리어

30단계 달빛 간판 5부품 구매와 최근 서비스 평점 4.5 이상을 모두 달성하면 5부품 순차 점등 뒤 결과 화면으로 이동합니다. 결과에는 방문 손님, 남은 돈, 평점과 오프라인을 제외한 활성 영업 시간을 표시합니다.

## 12. 검증

- 고정 seed 1,000개 × 30단계 × 3프로필
- 평균 P50 목표 오차 ±10%, 현재 최대 가격 보정 0.65%
- 모든 단계 `P90/P50 < 1.5`
- 후반 초보는 평균보다 25~40% 느리고 숙련은 20~30% 빠름
- FEVER 예상 매출 기여 10~17%
- 타입·경제/진행/피버/오프라인/저장 테스트·빌드·접근성 정적 감사 통과
- GitHub Pages Actions 빌드·배포와 공개 URL `https://junn0s.github.io/Meow/`의 HTTPS 200 응답 확인
- 신규 7~10단계 실브라우저 E2E는 2026-07-16 실행 환경 사용량 제한으로 보류
