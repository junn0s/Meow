# 냥포차: 달빛 아래 야식당 — 게임 설명서

## 1. 개요

| 항목 | 내용 |
| --- | --- |
| 영문명 | Meow Night Diner |
| 장르 | 경영 시뮬레이션·방치형 타이쿤·2D 픽셀아트 |
| 플랫폼 | 데스크톱·모바일 웹 브라우저 |
| 성장 길이 | 5챕터 × 30단계, 총 150단계 |
| 기술 | Phaser 3, TypeScript, Vite |
| 해상도 | 논리 480×270 / 렌더 960×540 |

비 오는 거리의 작은 포장마차를 운영해 6가지 야식을 팔고, 8명의 고양이 직원을 고용하며, 낮의 포근한 가게를 밤의 푸른 네온 명소로 성장시키는 경영 게임입니다.

## 2. 핵심 플레이 루프

```text
손님 입장 → 좌석 배정 → 주문 → 조리 → 서빙 → 식사 → 결제
        ↘ 인내심·이탈·평점 ↗
결제 수익 → 세부 구매 → 메뉴/좌석/직원/외형 성장 → 더 큰 수요
```

초반에는 사장 고양이가 직접 주문·조리·서빙을 담당합니다. 자동화 이후에도 사장님은 셰프와 별도의 수동 조리 슬롯 하나를 사용해 바쁜 메뉴를 지원할 수 있습니다. 어떤 메뉴, 좌석, 조리 슬롯과 서버 동선이 병목인지 판단하고 피버·홍보를 러시에 맞춰 운용합니다.

## 3. 조작과 접근성

| 입력 | 동작 |
| --- | --- |
| WASD·방향키 | 이동 |
| Space | 가까운 손님 주문, 해당 조리대에서 직접 조리, 완성 음식 들기, 일치 주문 서빙 |
| 마우스·터치 | 구매·버튼 |
| Esc | 일시정지·설정 |
| M | 음소거 |
| 모바일 조이스틱·ACTION | 연속 방향 이동·상호작용 |

모바일에서는 조이스틱을 끌어 대각선을 포함한 방향으로 연속 이동하며, 핀치·더블 탭 확대와 길게 누르기 선택 메뉴를 게임 화면 전체에서 차단합니다. 세로 화면은 포차 플레이 영역을 화면 중앙으로 내리고 기존 우측 성장 패널을 캔버스 아래의 큰 구매 카드로 옮깁니다. 모바일 조작 버튼은 최소 44×44px입니다. 상태는 색에만 의존하지 않고 `VIP`, `FEVER`, `RUSH`, `!`, `!!`, 주문 수량을 글자와 형태로 표시합니다. 운영체제 `prefers-reduced-motion` 또는 일시정지 메뉴의 움직임 줄이기를 사용하면 비·안개·증기·네온 전환과 결과 화면 반복 효과가 감소합니다.

## 4. 손님과 서비스

손님 상태는 다음 순서를 따릅니다.

```text
ENTERING → WAITING_FOR_SEAT → MOVING_TO_SEAT → ORDERING
→ WAITING_FOR_FOOD → EATING → PAYING → LEAVING
```

- 토끼: 기본 인내심 `×0.8`, 빠른 대응 필요
- 햄스터: 인내심 `×0.95`
- 강아지: 인내심 `×1.1`
- 너구리: 인내심 `×1.2`, 다량 주문 확률 증가
- VIP: 명성 Lv.4부터 등장, 인내심 `×1.25`, 기본 결제 `×2`, 별도 팁 없음
- 특별 주문: 명성 Lv.5부터 등장, 최소 2접시 주문과 결제 `×1.25`, 팁 확률 `+25%`

좌석 대기와 음식 대기는 서로 다른 인내심 예산을 사용합니다. 좌석 대기는 기본 18초 이상이며 앞에 기다리는 손님 한 명마다 4초를 추가합니다. 자리에 앉아 주문이 정해지면 남은 좌석 대기시간을 버리고 `기본 25초 이상 + 1인분 조리시간 ×1.5 + 추가 수량마다 조리시간 ×0.7 + 현재 주방 대기열 ×0.75`로 음식 인내심을 새로 계산합니다. 셰프가 없어 사장님이 직접 주문을 받아야 하면 5초 이상의 추가 여유를 줍니다. 인내심이 0이 되면 결제하지 않고 떠나고 콤보가 끊깁니다. 세 명이 연속 이탈하면 30초 동안 다음 손님 간격을 15% 늦추는 안전장치가 작동합니다. 평점은 최근 서비스 점수를 지수 가중 평균으로 반영하므로 예전 성공만으로 높은 평점을 영구 유지할 수 없습니다.

## 5. 메뉴와 주문

| 메뉴 | 해금 | 기본 가격 | 기본 조리시간 |
| --- | ---: | ---: | ---: |
| 어묵 | 1단계 | 18냥 | 4초 |
| 떡볶이 | 6단계 | 630냥 | 7초 |
| 순대 | 11단계 | 22K냥 | 10초 |
| 야식 라면 | 16단계 | 790K냥 | 13초 |
| 달빛 꼬치 | 21단계 | 28M냥 | 16초 |
| 달빛 정식 | 26단계 | 1B냥 | 20초 |

가격 레벨과 조리 속도 레벨은 분리됩니다. 가격은 지수 성장과 10/25/50/75/100레벨 이정표 배수를 사용하고, 조리시간은 최저 35% 바닥을 둡니다. 2·3개 주문은 추가 한 개당 기본 조리시간의 65%를 더합니다.

주문 수량 분포는 5단계 구간마다 강해져 26~30단계에는 1개 55%, 2개 35%, 3개 10%가 됩니다. 새 메뉴는 해금 직후 18% 가중치로 시작해 네 단계에 걸쳐 성숙 가중치로 올라갑니다.

## 6. 직원

| 역할 | 고용 단계 | 최대 인원 | 동작 |
| --- | --- | ---: | --- |
| 셰프 | 4/12/19/27/29 | 5명 | 주문자 한 명을 점유하고 해당 조리 티켓 완료까지 담당 |
| 서버 | 7/14/22/28/29 | 5명 | 완성 티켓 하나를 꺼내 정확한 주문자에게 왕복 배달 |

사장님이 직접 받은 주문은 해당 메뉴 조리대에서 다시 상호작용해야 조리가 시작됩니다. 사장님 수동 조리 슬롯 하나는 셰프 슬롯과 별개이므로 셰프가 떡볶이를 만드는 동안 사장님이 어묵을 동시에 만들 수 있습니다. 셰프 동시 조리는 메뉴별 1~3칸 조리대와 사용 가능한 셰프 중 작은 수로 제한되며, 같은 셰프가 두 칸을 동시에 점유하는 것은 금지합니다. 조리대 위에는 메뉴명과 현재 판매가격만 표시하고 내부 가격·속도·칸 레벨은 숨깁니다. 같은 티켓은 한 번만 완성 대기열에서 제거되고, 서버가 이동 중인 손님은 플레이어 수동 서빙 대상에서 제외됩니다. 손님이 떠나면 대기·조리·완성 티켓과 직원 점유를 함께 취소합니다.

예를 들어 셰프가 3명이고 어묵 조리대가 2칸이라면 셰프 2명이 어묵 두 접시를 동시에 만들고, 남은 셰프 1명은 다른 메뉴를 조리할 수 있습니다. 빈 셰프는 이미 접수된 미배정 조리 티켓을 우선 가져가며, 조리대에는 병렬 진행 중 `조리×2` 또는 `조리×3`이 표시됩니다.

직원 1~5번은 각각 빨강·파랑·초록·금색·달빛 분홍 모자띠/앞치마 또는 배지/트레이로 구분됩니다. 고용 인원만큼 포차 작업등도 늘어납니다.

셰프와 서버의 이동속도는 사장님과 동일한 초당 78픽셀입니다. 직원 속도 효과는 주문 접수·음식 인수 같은 작업시간만 줄이고 이동 자체를 순간이동처럼 빠르게 만들지 않습니다. 서버는 서빙을 마친 뒤 우측 대기열로 복귀하지 않고 마지막 서빙 위치에서 다음 작업을 기다립니다.

## 7. 결제·피버·홍보

```text
결제 = 메뉴 가격 × 수량 × VIP × 콤보 × FEVER/홍보
팁   = 위 기본 결제 × 팁률
```

- 콤보: 5/10/20회 연속 성공 시 `×1.05/×1.10/×1.15`
- 팁: 빠른 서비스 20%, 보통 서비스 10%, 늦은 서비스 0%
- FEVER Lv.1/2/3: 9/18/29단계 해금
- 피버 지속: 15/18/20초, 결제 `×1.5/×1.65/×1.8`
- 피버 조리 속도: `×1.15/×1.25/×1.35`
- 피버 직원 속도: `×1.12/×1.20/×1.30`, 팁 확률 `+8/+12/+18%`
- 메뉴 홍보: 30초 동안 대상 메뉴 주문 가중치 `×2`, 결제 `×1.25`

피버는 결제할 때 확정 게이지를 채우고 100%에서 자동 발동하므로 확률 상자를 사용하지 않습니다. 발동 중 청록 네온·반사와 초당 3.2회의 피버 리듬 레이어가 함께 강해지며, 종료 3초 전에 HUD 색상·토스트·카메라 플래시로 경고합니다. 원곡 재생 속도는 바꾸지 않아 BGM 길이와 시간대가 끝까지 동기화됩니다. 레벨별 시뮬레이션 매출 기여 목표는 10~17%입니다.

명성은 5단계 성장 묶음에 대응하는 Lv.1~6 등급입니다. 결제 보너스는 레벨마다 2%씩 올라 Lv.6에서 `+10%`, 팁 확률도 같은 비율로 증가합니다. Lv.2에서 햄스터, Lv.3에서 너구리, Lv.4에서 VIP, Lv.5에서 특별 주문이 해금됩니다. VIP 확률은 Lv.4/5/6에서 `4/7/11%`, 특별 주문은 Lv.5/6에서 `10/18%`입니다. 고급 외관과 운영 시설에도 명성 요구치가 있으며 일시정지의 `명성·피버` 패널에서 현재 수치를 확인할 수 있습니다.

## 8. 난이도와 5개 챕터·150단계

각 챕터는 독립된 30단계 성장 곡선을 사용합니다. 챕터 1은 달빛 골목 포차, 챕터 2는 해변 칵테일 바, 챕터 3은 한옥 한식당, 챕터 4는 클래식 양식 레스토랑, 챕터 5는 편백 카운터의 일식 오마카세입니다. 챕터마다 여섯 메뉴의 이름·음식 픽셀아트·조리대·간판·배경 팔레트·맵 장식·최종 시설이 모두 바뀝니다.

30단계와 평점 4.5를 완료하면 다음 챕터는 1단계, 0냥, 평점 1.0, 직원 0명, 기본 메뉴 한 개와 메뉴별 조리대 한 칸으로 시작합니다. 털색과 의상, 공용 시설 상점 구매 내역은 장기 수집 보상으로 유지됩니다.

| 구간 | 단계당 목표 | 운영 변화 |
| --- | ---: | --- |
| 1~5 | 30초~2분 | 규칙 학습, 첫 셰프 |
| 6~10 | 3분 24초→2분 55초 | 떡볶이, 첫 서버, FEVER Lv.1 |
| 11~15 | 5분 36초 | 순대, 직원 2명, 다량 주문 |
| 16~20 | 8분 24초 | 라면, FEVER Lv.2, 높은 주방 압력 |
| 21~25 | 11분 54초 | 꼬치, VIP, 러시 시작 |
| 26~30 | 14/17.5/21/24.5/42분 | 정식, 직원 4명, 최종 복합 병목 |

25~30단계는 10초 예고 뒤 25~30초 러시가 발생합니다. 러시는 자홍색 경고와 `RUSH`, 피버는 청록색 점등과 `FEVER`로 표시됩니다. 마지막 30단계는 8분 24초 간격의 달빛 간판 5부품으로 나뉘며 각 부품은 수익 `×1.12`와 점등 연출을 제공합니다.

## 9. 시간대와 디자인

활성 플레이의 낮→노을→밤→새벽 길이는 현재 재생되는 각 BGM의 실제 길이와 같습니다. 요청한 6개 음악 루프가 끝나면 첫 루프로 돌아갑니다. 일시정지·튜토리얼·메뉴·오프라인 시간은 시계를 진행시키지 않습니다.

- 낮: 높은 가독성의 크림·주황 포차
- 노을: 따뜻한 하늘과 약한 반사
- 밤: 군청 도시, 청록 네온, 자홍 포인트, 젖은 바닥 반사
- 새벽: 보랏빛 하늘과 낮은 안개

밤 진입 시 최대 24개 네온이 약 1.2초 동안 순서대로 켜집니다. 분위기 파티클은 비 48개와 안개 12개로 구성한 60개 고정 풀이며 매 프레임 새 객체를 만들지 않습니다. `성능 설정`은 글자용 2배 해상도를 항상 유지하면서 다음 세 프로필을 즉시 적용하고 저장합니다.

| 모드 | 프레임 | 조명·날씨 | 반사·업데이트 |
| --- | ---: | --- | --- |
| 고화질 | 60FPS | 조명 24·비 48·안개 12 | 반사 켜짐·매 프레임 |
| 균형 | 모바일 30FPS/PC 60FPS | 모바일 조명 16·비 24·안개 6 | 반사 켜짐·연출 주기 제한 |
| 절전 | 24FPS | 조명 8·비 12·안개 2 | 반사 꺼짐·연출 80ms/UI 200ms |

보이지 않는 직원과 화면 밖 손님의 애니메이션 텍스처 갱신은 건너뛰고, 탭이 숨겨지면 전체 게임 로직·음악·Web Audio를 정지합니다. `모션 최소화`는 성능 모드와 별도로 사용할 수 있습니다.

## 10. 오프라인과 저장

- 자동 저장은 4초마다 실행되며 일시정지 화면에 마지막 저장 시각을 표시한다.
- 셰프와 서버가 모두 있을 때만 자동 운영되며 주방, 서빙, 좌석, 손님 유입 처리량 중 가장 느린 값을 실제 처리량으로 삼는다.
- 명성에 따라 오프라인 효율이 25%에서 최대 65%까지 성장하고, 복귀 시 최대 4시간을 별도 정산창으로 보여준다.

### 메뉴별 조리대와 동시 조리

- 해금 메뉴마다 기본 조리대 1칸을 보유하고 시설 상점에서 2칸, 3칸으로 확장한다.
- 셰프의 실제 동시 조리 수는 메뉴별 조리대 슬롯과 사용 가능한 셰프 중 작은 값이며, 사장님은 별도의 수동 조리 슬롯 하나를 가진다.
- 사장님이 직접 받은 주문은 해당 메뉴 조리대에서 조리 시작 입력을 해야 한다.
- 수량 2·3 주문도 한 접시씩 티켓으로 나눠 여분 셰프가 함께 조리할 수 있고, 완성된 접시는 다른 주문 사이에 자유롭게 섞어 서빙한다.

오프라인 정산은 돈만 지급하며 구매와 단계 진행을 자동 실행하지 않습니다. 명성과 심야 장부 시설이 자동운영 효율을 올리고, 정산창에는 경과 시간·효율·현재 병목을 함께 표시합니다.

`SaveData v5`는 현재 챕터, 돈, 평점, 손님, 30단계 진행, 메뉴, 직원, 피버, 시간대, 활성 시간, 전체·배경음·효과음 설정, 성능·움직임 설정, 전체 클리어 상태를 저장합니다. 영업 중 자동 저장과 수동 저장을 모두 지원하며 v1~v4 저장은 챕터 1 진행으로 안전하게 이전됩니다. 메뉴별 조리대 슬롯은 챕터마다 별도로 저장됩니다.

메인·낮·노을·밤·새벽은 각각 별도의 로파이 재생목록을 사용합니다. 원본 MP3는 배포용 Opus 80kbps로 변환해 모바일 전송량을 줄였습니다. 일시정지 메뉴의 `SOUND MIXER`에서 전체·배경음·효과음을 10% 단위로 조절하거나 각각 음소거할 수 있으며 설정은 즉시 저장됩니다. 자동재생이 제한된 브라우저에서는 첫 사용자 조작 뒤 재생을 시작하고, 탭이 숨겨지면 HTML 오디오와 Web Audio 컨텍스트를 모두 정지했다가 복귀 시 이어서 재생합니다.

`MOONLIGHT WARDROBE`는 털색 4종, 눈 3종, 모자 4종, 앞치마 4종, 장식 4종을 제공한다. 구입·장착 즉시 상점 미리보기와 실제 이동 캐릭터에 픽셀 레이어로 반영하며 소유 목록과 조합을 브라우저에 영구 저장한다. 코스메틱에는 능력치를 붙이지 않아 원하는 외형을 자유롭게 선택할 수 있다.

`MOONLIGHT SHOP`은 주방·홀·외관·운영·직원의 5개 탭과 총 16개 시설을 제공한다. 조리 속도, 셰프·서버 속도, 인내심, 좌석, VIP, 결제, 명성, 팁, 피버 충전, 오프라인 효율을 각각 명시된 수치만큼 바꾼다. 청사초롱은 VIP, 푸른 천막은 인내심, 달빛 네온은 야간 팁, 초승달 간판은 명성·손님 유입·특별 주문 확률을 높인다. 가게 시설은 구매 직후 포차 장식에도 픽셀 오브젝트로 나타나며, 청록 셰프복·자홍 서버복은 실제 직원 외형을 바꾸고 달빛 직원 배지는 양쪽 직원 속도를 5% 높인다.

## 11. 클리어

각 챕터의 30단계 최종 시설 5부품과 최근 서비스 평점 4.5 이상을 달성하면 챕터 결과 화면으로 이동합니다. 챕터 1~4는 다음 콘셉트의 식당을 열며 경제와 직원 진행을 초기화하고, 챕터 5 완료 후에만 전체 게임 엔딩을 표시합니다.

## 12. 검증

- 고정 seed 1,000개 × 30단계 × 3프로필
