## 2. 한 줄 소개

비 오는 밤의 작은 포장마차에서 동물 손님에게 야식을 팔고, 메뉴·좌석·고양이 직원을 해금해 달빛 아래 최고의 야식당으로 성장시키는 짧고 따뜻한 경영 게임입니다.

## 3. 배경과 플레이어 경험

길고양이 한 마리가 어묵 냄비와 작은 좌석만 있는 포장마차를 엽니다. 처음에는 사장 고양이가 주문 접수부터 음식 전달까지 직접 맡습니다. 손님에게 음식을 제때 제공해 돈과 좋은 평가를 얻고, 시설을 하나씩 확장하면 떡볶이와 붕어빵을 팔 수 있습니다. 셰프와 서버를 고용한 뒤에는 반복 작업이 자동화되고, 플레이어는 가게의 성장과 동선 관리에 집중하게 됩니다.

게임이 전달하려는 감성은 다음과 같습니다.

- 짙은 남색 밤과 따뜻한 주황빛 포장마차의 대비
- 빗방울, 음식의 김, 작은 하트와 동전이 만드는 잔잔한 움직임
- 토끼, 강아지, 햄스터, 너구리 손님이 주는 친근함
- 짧은 시간에도 분명하게 느껴지는 수동 운영에서 자동 운영으로의 성장

## 4. 핵심 게임 루프

```text
손님 등장
→ 빈 좌석 예약 및 착석
→ 플레이어가 주문 접수
→ 해금된 조리대에서 음식 조리
→ 플레이어 또는 서버가 음식 전달
→ 손님 식사
→ 돈과 만족도 획득
→ 다음 시설·메뉴·직원 구매
→ 더 빠르고 큰 가게로 성장
```

초반의 직접 조작은 게임 규칙을 학습시키고, 중후반의 직원 자동화는 성장의 보상이 됩니다. 돈을 버는 속도와 다음 업그레이드 비용을 조절해 첫 구매는 1분 이내, 최종 클리어는 5~8분 안에 도달하도록 설계합니다.

## 5. 조작과 상호작용

| 입력 | 동작 |
| --- | --- |
| `WASD` 또는 방향키 | 사장 고양이 이동 |
| `Space` | 가장 가까운 유효 대상과 상호작용 |
| 마우스 | 업그레이드·메뉴·설정 버튼 클릭 |
| `Esc` | 일시 정지 또는 설정 열기·닫기 |
| `M` | 효과음 켜기·끄기 |

`Space`의 동작은 주변 상태에 따라 결정됩니다.

1. 주문 아이콘이 있는 손님 곁에서는 주문을 접수합니다.
2. 완성 음식이 있는 조리대 곁에서는 음식 하나를 듭니다.
3. 들고 있는 음식과 주문이 일치하는 손님 곁에서는 서빙합니다.

현재 들고 있는 음식, 상호작용 가능한 대상, 필요한 키는 HUD와 말풍선으로 표시합니다. 업그레이드 구매는 마우스 클릭으로 분리해 이동 중 실수로 구매하는 일을 줄입니다.

## 6. 손님 시스템

손님은 토끼, 강아지, 햄스터, 너구리의 네 외형 중 하나로 등장합니다. MVP에서 캐릭터별 차이는 선호 메뉴와 시각적 표현에 집중하고, 핵심 상태 규칙은 공통으로 사용합니다.

```text
ENTERING
→ WAITING_FOR_SEAT
→ MOVING_TO_SEAT
→ ORDERING
→ WAITING_FOR_FOOD
→ EATING
→ PAYING
→ LEAVING
```

- 입구에서 생성된 손님은 빈 좌석을 찾습니다.
- 좌석을 선택한 즉시 예약해 동시에 생성된 손님이 같은 자리를 선택하지 않게 합니다.
- 빈 좌석이 없으면 입구 대기 위치에서 기다립니다.
- 착석 후 해금된 메뉴 중 하나를 주문합니다. 선호 메뉴가 해금되어 있으면 선택 확률을 높일 수 있습니다.
- 주문 접수 또는 음식 대기가 길어질수록 인내심과 예상 팁이 감소합니다.
- 식사가 끝나면 돈을 남기고 출구로 이동하며, 화면 밖에 도착하면 객체와 좌석 점유를 정리합니다.

### 만족도와 팁

| 만족도 | 결제 결과 |
| --- | --- |
| 매우 만족 | 영업 보너스 적용 정산액 + 30% 팁 |
| 만족 | 영업 보너스 적용 정산액 + 10% 팁 |
| 보통 | 영업 보너스 적용 정산액 |
| 불만족 | 영업 보너스 적용 정산액, 팁 없음 |
| 매우 불만족 | MVP에서는 주문 취소 대신 팁 없음으로 단순화 |

평점은 성공적으로 응대한 손님의 만족도를 누적해 계산합니다. 빠른 주문·서빙은 수익과 평점을 함께 높이며, 평점 5점은 최종 클리어 조건 중 하나입니다.

## 7. 메뉴와 조리

| 메뉴 | 기본 조리 시간 | 메뉴판 가격 | 설명 |
| --- | ---: | ---: | --- |
| 어묵 | 3초 | 10냥 | 처음부터 판매하는 기본 메뉴 |
| 떡볶이 | 5초 | 25냥 | 중반 조리대를 구매하면 해금되는 고수익 메뉴 |
| 붕어빵 | 7초 | 45냥 | 후반 기계를 구매하면 해금되는 최고가 메뉴 |

주문은 조리 대기열에 저장하며, 메뉴 데이터가 조리 시간·가격·해금 조건의 단일 기준이 됩니다. 조리가 끝나면 조리대에 완성 표시가 나타납니다. 플레이어는 음식을 하나만 운반할 수 있고, 주문이 일치할 때만 손님에게 전달할 수 있습니다.

셰프 고양이를 고용하면 접수된 주문을 자동으로 조리합니다. 서버 고양이를 고용하면 완성 음식 중 해당 손님의 주문을 찾아 자동 전달합니다. 직원이 활성화되어도 플레이어의 수동 상호작용은 유지해 자동화 대기 중 병목을 직접 해소할 수 있게 합니다.

### 밤손님 영업 보너스

메뉴판 가격은 원 기획의 10냥, 25냥, 45냥을 유지합니다. 다만 해커톤 MVP의 5~8분 성장 목표를 맞추기 위해 실제 결제의 기본 정산액에는 `밤손님 영업 보너스 ×3`를 적용합니다.

```text
기본 정산액 = 메뉴판 가격 × 3
팁 = 기본 정산액 × 만족도 팁률(30%, 10% 또는 0%)
최종 수익 = 기본 정산액 + 반올림한 팁
```

이 보너스는 숨은 배율로 두지 않고 시작 튜토리얼과 결제 피드백에 표시합니다. 업그레이드 가격은 바꾸지 않아 첫 구매 1분 이내, 최종 클리어 5~8분이라는 MVP 템포를 맞춥니다.

## 8. 돈과 10단계 업그레이드

모든 구매는 경제 시스템이 현재 돈, 선행 단계, 중복 구매 여부를 확인한 뒤 한 번만 처리합니다. UI는 결과를 표시하지만 돈을 직접 변경하지 않습니다.

| 단계 | 이름 | 가격 | 효과 |
| ---: | --- | ---: | --- |
| 1 | 어묵 조리대 | 무료 | 어묵 조리 및 판매 가능 |
| 2 | 추가 좌석 | 50냥 | 좌석 2개 추가 |
| 3 | 빠른 냄비 | 100냥 | 모든 음식 조리 시간 30% 단축 |
| 4 | 떡볶이 조리대 | 200냥 | 떡볶이 주문·조리 가능 |
| 5 | 셰프 고양이 | 350냥 | 접수된 주문의 조리 자동화 |
| 6 | 추가 테이블 | 500냥 | 좌석 2개 추가 |
| 7 | 붕어빵 기계 | 700냥 | 붕어빵 주문·조리 가능 |
| 8 | 서버 고양이 | 900냥 | 완성 음식의 서빙 자동화 |
| 9 | 네온 간판 | 1,200냥 | 손님 생성 간격 30% 감소 |
| 10 | 달빛 간판 | 2,000냥 | 가게 완성, 최종 클리어 판정 활성화 |

업그레이드는 정해진 순서로 노출합니다. 돈이 부족한 단계는 비활성화하고, 구매가 끝난 단계는 완료 상태로 표시합니다. 좌석·조리대·직원·간판 변화는 구매 직후 게임 화면에 반영되어 성장 보상을 시각적으로 확인할 수 있어야 합니다.

## 9. 저장과 이어하기

진행 데이터는 서버 없이 브라우저 `localStorage`에 저장합니다.

```ts
interface SaveData {
  version: number;
  money: number;
  purchasedUpgradeIds: string[];
  customerCount: number;
  rating: number;
  settings: GameSettings;
  muted: boolean;
  tutorialCompleted: boolean;
  playStartedAt: number;
  elapsedMs: number;
  cleared: boolean;
  lastSavedAt: number;
}
```

저장은 구매·수익·평점처럼 중요한 진행이 바뀐 직후, 4초 간격, 장면 종료 시 수행합니다. 시작 화면에서 저장 데이터를 읽고, 스키마 버전과 값의 유효성을 확인한 뒤 돈, 해금 메뉴, 좌석, 직원, 간판을 같은 순서로 복원합니다. JSON 파싱 실패나 범위를 벗어난 값이 있으면 오류로 멈추지 않고 새 게임 기본값으로 복구합니다.

`elapsedMs`는 게임 장면이 실제로 진행되는 시간만 누적하므로 메뉴, 3단계 튜토리얼, 일시 정지와 오프라인 시간은 결과의 영업 시간에서 제외됩니다. 브라우저 `localStorage`와 메모리 fallback이 서로 다른 저장본을 갖고 있으면 두 값의 JSON·필드 유효성을 확인하고 `lastSavedAt`이 최신인 값을 사용합니다. 새 게임 초기화에서는 진행을 지우되 사용자가 선택한 음소거 설정을 보존합니다.

## 10. 클리어 조건

아래 두 조건을 모두 만족하면 게임을 클리어합니다.

1. 10단계 `달빛 간판` 구매
2. 가게 평점 5점 달성

조건을 만족하면 손님과 조리 타이머를 안전하게 정리하고 클리어 연출과 결과 화면을 표시합니다. 결과 화면에는 최종 자산, 응대한 손님 수, 획득 평점과 플레이 시간을 보여주며 새 게임 또는 이어하기 경로를 제공합니다.

## 11. 화면 구성

### 시작 화면

- 게임 제목과 한 줄 설명
- 새 게임·이어하기 버튼
- Enter 시작과 핵심 조작 안내
- 저장 데이터 초기화와 사운드 설정

### 게임 화면

- 상단 HUD: 보유 금액, 평점, 방문 손님 수, 현재 들고 있는 음식
- 중앙 플레이 공간: 조리대, 좌석, 손님, 플레이어와 직원
- 손님 머리 위: 주문 음식, 인내심, 만족 표현
- 화면 가장자리: 상호작용 안내와 다음 업그레이드 패널
- 첫 실행 오버레이: 이동, 주문·정산, 시설 성장의 3단계 튜토리얼

### 결과 화면

- 달빛 간판 완성 연출
- 최종 통계
- 다시 시작 또는 이어하기 선택

## 12. 시각·음향 제작 방식

게임의 이미지 파일과 음원 파일은 외부에서 가져오지 않습니다.

- 캐릭터, 음식, 가구, 간판, UI 아이콘은 Phaser Graphics 또는 Canvas API로 작은 색 블록을 그린 뒤 텍스처로 생성합니다.
- 게임 월드는 `480 × 270` 논리 좌표계를 유지하면서 실제 캔버스는 `960 × 540`으로 렌더링합니다. 글자는 2배 내부 해상도로 래스터화하고, 픽셀아트는 `pixelArt: true`, `roundPixels: true`, NEAREST 필터를 유지합니다. 작은 화면에서는 16:9 비율로 축소합니다.
- 비, 음식의 김, 하트, 동전 효과는 단순 도형과 파티클로 표현합니다.
- 동전, 조리 완료, 구매, 입장, 해금, 클리어 효과음은 Web Audio API의 oscillator, gain, envelope 조합으로 실행 시 합성합니다.
- 외부 BGM은 사용하지 않습니다. BGM을 나중에 추가한다면 출처와 라이선스를 별도로 기록해야 합니다.

자세한 내용은 [에셋 및 라이선스](asset-licenses.md)에 기록합니다.

## 13. 기술 설계

```text
Boot / Menu / Game / Result Scene
              │
        입력·표시 조정
              │
 ┌────────────┼───────────────────────────┐
 │            │                           │
Entity     Game Systems                  UI
플레이어    손님·좌석·주문·조리           HUD
손님        경제·업그레이드·저장           업그레이드 패널
직원                                      안내·결과 화면
 │            │                           │
 └────────────┴──── 데이터 정의 ──────────┘
                   │
          메뉴 / 손님 / 업그레이드
```

시스템별 핵심 원칙은 다음과 같습니다.

- 좌석은 검색 시점이 아니라 선택 시점에 예약해 중복 배정을 막습니다.
- 손님 상태 전환과 좌석 해제는 한 경로에서 처리해 유령 점유를 막습니다.
- 조리 타이머는 장면 일시 정지·종료 때 함께 관리합니다.
- 결제와 구매는 경제 시스템만 돈을 변경하도록 합니다.
- 업그레이드 UI는 데이터와 현재 상태를 읽어 표시하고 규칙을 중복 구현하지 않습니다.
- 저장 복원은 구매 효과를 재적용하되 돈을 다시 차감하지 않습니다.

## 14. 목표 성능과 검증 기준

- 배포 URL 접속 후 10초 이내에 게임 진입
- 첫 손님 15초 이내 등장
- 첫 업그레이드 1분 이내 구매 가능
- 한 판 5~8분 안에 최종 클리어 가능
- 여러 손님이 같은 좌석에 배정되지 않음
- 존재하지 않는 메뉴가 주문되지 않음
- 음식이 다른 손님에게 잘못 서빙되지 않음
- 동일 업그레이드가 중복 결제되지 않음
- 새로고침 후 돈과 업그레이드 효과가 함께 복원됨
- 잘못된 저장 데이터로 게임이 중단되지 않음
- GitHub Pages 하위 경로에서도 모든 코드와 생성 리소스가 로드됨
- 키보드를 누른 채 창 포커스를 잃어도 이동 상태가 고착되지 않음

## 15. MVP 제외 범위

온라인 멀티플레이, 계정·로그인, 서버 데이터베이스, 결제, 랭킹, 여러 맵, 복잡한 인벤토리, 모바일 터치 조작과 무한 진행은 구현 범위에 포함하지 않습니다.
