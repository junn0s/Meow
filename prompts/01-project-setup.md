# 01. 프로젝트 기반 설정 프롬프트

## 목적

Phaser 3 + TypeScript + Vite 기반의 실행 가능한 게임 뼈대를 만들고, 내부 해상도·입력·장면·코드 생성 리소스 정책을 먼저 고정한다.

## AI에 전달할 프롬프트 전문

```text
빈 저장소에 웹 기반 2D 픽셀아트 경영 시뮬레이션 게임 프로젝트를 구현해 줘.

게임 제목은 "냥포차: 달빛 아래 야식당", 영문명은 "Meow Night Diner"다. Phaser 3, TypeScript, Vite, HTML/CSS를 사용하고 GitHub Pages 배포를 고려한다.

이번 작업 범위는 프로젝트 기반, 기본 장면, 플레이어 이동, UI 뼈대, 코드 생성 리소스다. 아직 복잡한 손님·주문·업그레이드 규칙을 한 파일에 임시 구현하지 말고 다음 작업에서 확장할 수 있는 책임 구조를 만든다.

필수 기술 요구사항:

1. TypeScript strict 모드를 사용하고 any 타입을 사용하지 않는다.
2. 게임 내부 해상도는 480×270이다.
3. Phaser 설정에 pixelArt: true와 roundPixels: true를 적용한다.
4. 브라우저 크기가 달라져도 16:9 비율을 유지하고 가능한 범위에서 정수 배율로 확대한다.
5. 배경색이 보이는 letterbox는 허용하되 게임 화면이 잘리거나 비율이 찌그러지면 안 된다.
6. BootScene, MenuScene, GameScene, ResultScene을 분리한다.
7. BootScene은 코드 생성 텍스처와 공통 초기화만 담당한다.
8. MenuScene은 제목, 새 게임, 이어하기, 조작 안내를 표시한다.
9. GameScene은 시스템을 연결하되 손님·경제 규칙을 장면 내부에 모두 넣지 않는다.
10. ResultScene은 최종 통계와 다시 시작 경로를 제공할 수 있는 구조로 만든다.
11. 치즈 고양이 플레이어를 외부 이미지 없이 코드로 그린 픽셀 텍스처로 표시한다.
12. WASD와 방향키로 이동하고 맵 경계를 벗어나지 못하게 한다.
13. Space는 상호작용, Esc는 일시 정지/설정에 사용할 입력으로 등록한다.
14. 마우스로 메뉴와 업그레이드 UI를 클릭할 수 있는 공통 버튼 방식을 준비한다.
15. 키를 누른 상태에서 창 포커스를 잃어도 이동이 고착되지 않게 처리한다.
16. HUD에 최소한 돈, 평점, 방문 손님 수, 현재 운반 음식 영역을 둔다.
17. 돈의 초기값과 게임 상수는 데이터 또는 명시적 설정에서 관리한다.
18. localStorage 저장 시스템이 추가될 자리를 마련하되, 읽기 실패가 게임 시작을 막지 않게 설계한다.

에셋 정책:

1. 외부 PNG, SVG, GIF, 타일맵, 아이콘, 웹폰트, 음원 파일을 추가하거나 다운로드하지 않는다.
2. 캐릭터, 가구, 음식, 배경, UI 아이콘은 Phaser Graphics 또는 Canvas API로 정수 좌표의 색상 블록을 그려 텍스처로 생성한다.
3. 비, 김, 하트, 동전은 코드 생성 도형 또는 파티클로 표현할 수 있게 한다.
4. 효과음은 Web Audio API의 oscillator, gain, envelope를 사용해 런타임 합성한다.
5. 브라우저 자동 재생 정책에 맞게 첫 키 입력이나 클릭 후 AudioContext를 resume한다.
6. 외부 에셋 URL에 런타임 요청을 보내지 않는다.

프로젝트 품질 요구사항:

1. 기능별 파일 분리를 유지한다.
2. 공유 상태를 무분별한 전역 변수로 두지 않는다.
3. Scene 종료 시 등록한 키·포인터·resize·timer 이벤트를 정리할 수 있게 한다.
4. 화면 텍스트는 한국어로 작성한다.
5. 색상 대비를 확보하고 작은 내부 해상도에서도 HUD가 겹치지 않게 한다.
6. npm run dev, npm run build, npm run preview 스크립트를 제공한다.
7. Vite base는 GitHub Pages의 저장소 하위 경로에서 동작하도록 설정한다. 로컬 루트 경로를 하드코딩하지 않는다.
8. 소스 맵이나 개발 디버그 표시가 프로덕션 화면에 노출되지 않게 한다.

구현이 끝나면 다음을 보고해 줘.

1. 생성하거나 수정한 파일 목록과 각 파일의 책임
2. 실행 명령
3. 외부 에셋을 사용하지 않았음을 확인한 방법
4. TypeScript와 build 실행 결과
5. 아직 구현하지 않은 손님, 주문, 조리, 업그레이드, 저장, 클리어 항목

명령을 실제로 실행하지 않았다면 성공했다고 쓰지 말고 "검증 예정"이라고 표시해 줘. 기존 파일이 있다면 먼저 확인하고 사용자 변경을 덮어쓰지 마.
```

## 기대 검증

- 개발 서버와 preview에서 같은 장면 흐름이 보인다.
- 480×270 화면이 찌그러지지 않고 픽셀 경계가 흐려지지 않는다.
- WASD와 방향키가 동시에 등록돼도 이동 속도가 비정상적으로 빨라지지 않는다.
- 외부 이미지·음원 네트워크 요청이 없다.
- 장면을 반복 전환해도 이벤트가 중복 등록되지 않는다.
